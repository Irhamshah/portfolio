AAC Messenger

Role Full-Stack Developer & Co-Founder
Timeline 9 months (on going)
Platform iOS & Android
View Live App Visit Website

Overview

AAC Messenger is a mobile application designed to enhance communication and accessibility for non-verbal special needs individuals. By integrating and digitizing existing solutions into a single, user-friendly platform, the app aims to empower individuals to communicate more effectively with their caregivers.

Developed alongside my brother as part of our startup journey, this project represents our commitment to creating technology that truly makes a difference. Through extensive user research and iterative design, we've created an app that prioritizes simplicity without sacrificing functionality.

The Problem

Many existing accessibility apps are either too complex, lacking key features, not designed with actual user feedback or simply too expensive. Users often face:

  • Steep learning curves that create barriers instead of removing them
  • Limited customization options that don't adapt to individual requirements
  • High cost, leading to many not being able to access them

The Solution

We designed an assistive communication app that provides a fast, simple, and accessible way for non-verbal users to express themselves:

  • Grid-Based Communication Board: A clean, customizable grid of cards that users can tap to speak words or phrases aloud.
  • Folder Navigation System: Cards can function as folders, allowing users to navigate into categories (e.g., Food, Emotions, Actions) for more specific vocabulary.
  • Text-to-Speech Output: Every card instantly speaks its content, enabling quick and natural communication.
  • Personalized Vocabulary: Users can store cards and folders created by the admin or tailor them based on their own needs.
  • Real-Time Messaging: Built-in chat system that lets caregivers and users communicate seamlessly.
  • Accessible Design: Large touch targets, colorful cards, and a minimal interface optimized for users with autism and other communication challenges.

"Our goal was simple: make every interaction feel natural and effortless, removing barriers instead of adding complexity."

Key Features

  1. Folder-Based Navigation – Cards can act as folders, allowing users to navigate deeper into categories like Food, Places, Actions, or Emotions.
  2. Customizable Vocabulary – Users can access cards created by the admin or tailored for their needs, making the app adaptable to different communication styles.
  3. Built-in Chat System – Real-time messaging allows users and caregivers to communicate seamlessly within the app.

Technical Implementation

Built with React Native and Expo for cross-platform performance, the app uses a modern, scalable backend architecture to support real-time communication and a smooth user experience:

  • Firebase Authentication for secure and seamless user sign-in
  • WebSockets + DynamoDB for fast, reliable real-time messaging
  • Expo Speech & Accessibility APIs for text-to-speech and an easy-to-use interface
  • Local Caching for quick loading of cards, folders, and recent messages
  • Lightweight Architecture optimized for responsiveness and accessibility-focused interactions

Challenges & Learnings

Challenge: Designing an interface that is simple enough for non-verbal users, yet flexible enough to handle folders, categories, and real-time communication. Traditional mobile UI patterns didn’t work well for users with autism or limited motor control.

Solution: I redesigned the grid multiple times—adjusting card sizes, spacing, colors, and navigation flow based on how my younger brother interacted with it. I also simplified touch interactions, added clearer visual hierarchy, and refined the folder system to minimize cognitive load.

Learning: Accessibility isn’t about adding features—it’s about removing friction. Every iteration taught me how differently people communicate, and how thoughtful UI decisions can dramatically improve ease of use for someone who relies on the app every day.
