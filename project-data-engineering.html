Enterprise Project

Enterprise Data Pipeline Automation

Role Data Engineer
Company Accenture Singapore
Timeline 14 months
Project Multi-million dollar Government project

Overview

Served as a Data Engineer contractor at Accenture Singapore, leading development efforts on a large-scale data analytics infrastructure project. Focused on automation, pipeline development, and team leadership to deliver enterprise-grade data solutions for a major client.

The project involved building automated ETL pipelines, creating data warehousing solutions, and developing custom transformation jobs to process massive datasets. As the main developer for Wave 1, I was responsible for architecting solutions that would scale across multiple project phases.

Project Context

Part of a multi-wave, multi-million dollar enterprise transformation project. The client needed to modernize their data infrastructure, moving from legacy systems to a cloud-based analytics platform capable of handling diverse data sources and complex business requirements.

Led a development team within the larger Data Analytics organization, coordinating with stakeholders, architects, and business analysts to deliver production-ready data pipelines on schedule.

Automation Solutions

Developed comprehensive automation systems to eliminate manual processes and reduce deployment time from days to minutes:

AWS Glue Job Automation

  • Automated Creation: Built Python scripts that read RDS metadata and automatically generate AWS Glue jobs, CloudFormation JSON, and Python ETL scripts
  • Template System: Created reusable templates that adapt to different data sources and transformation requirements
  • S3 Deployment: Automated storage of generated scripts in S3 buckets with proper versioning and organization
  • Configuration Management: Metadata-driven approach allows new pipelines to be configured without code changes

Redshift Schema Automation

  • SQL Generation: Developed tools to automatically generate Redshift table creation scripts from RDS metadata stored in JSON format
  • Data Type Mapping: Intelligent mapping between source database types and optimal Redshift column types
  • Index Optimization: Automatic generation of distribution and sort keys based on query patterns
  • Schema Validation: Built-in checks to ensure generated schemas meet data warehouse best practices

Result: Reduced pipeline deployment time from 2-3 days of manual work to under 1 hour of automated execution, while eliminating human error.

Pipeline Development

As team lead, directed the development of complex ETL pipelines handling diverse data sources:

  • Connected to multiple source systems including relational databases, APIs, and flat files

  • Built error handling and retry mechanisms for resilient data extraction
  • Created monitoring and alerting for failed extractions

Data Transformation

Data Loading

  • Implemented efficient bulk loading into AWS Redshift data warehouse

Custom Metrics Development

Worked directly with business stakeholders to understand requirements and develop custom Glue jobs that calculate client-specific metrics:

  • Business Logic Translation: Converted complex business requirements into efficient SQL and Python transformations
  • Aggregation Jobs: Created jobs that pre-compute aggregated metrics for faster downstream reporting
  • Data Quality Metrics: Built jobs to track data completeness, accuracy, and timeliness
  • Performance Tuning: Optimized metric calculations to run efficiently on large datasets

Orchestration with Step Functions

Developed AWS Step Functions to orchestrate complex data workflows with proper error handling and dependencies:

  • Pipeline Orchestration: Created state machines that run Glue jobs in the correct sequence with proper dependencies
  • Error Handling: Implemented retry logic, error branching, and notification systems for pipeline failures
  • Parallel Processing: Designed workflows that process independent data streams in parallel for faster completion
  • Monitoring Integration: Integrated with CloudWatch for real-time pipeline monitoring and alerting

Challenges & Solutions

Challenge: Manual creation of 50+ similar Glue jobs was time-consuming, error-prone, and difficult to maintain consistently.

Solution: Built metadata-driven automation that generates jobs from configuration. One change to the template immediately applies to all generated jobs, and new pipelines can be added by updating metadata without writing code.

Challenge: Complex dependencies between data pipelines made orchestration difficult, with frequent failures due to timing issues.

Solution: Implemented Step Functions with explicit dependency management, retry logic, and parallel processing where possible. Added comprehensive error handling and alerting so issues could be resolved quickly.

Challenge: Client requirements for custom metrics were complex and frequently changed during development.

Solution: Created a flexible framework for metric calculation that could accommodate changes without major refactoring. Worked closely with business analysts to clarify requirements early and validate outputs regularly.
